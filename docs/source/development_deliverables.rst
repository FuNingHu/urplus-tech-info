Development Deliverables
========================

This section outlines the required deliverables for UR+ product certification.

Overview
--------

Partners must provide a complete set of deliverables to ensure their product meets UR+ standards and can be properly evaluated for certification.

Hardware Deliverables
---------------------

For hardware products, the following items are required:

1. **Physical Product Samples**

   * Minimum 2 units for testing
   * Production-representative quality
   * Complete with all accessories and cables

2. **Technical Specifications**

   * Mechanical drawings and dimensions
   * Electrical specifications
   * Weight and center of gravity data
   * Environmental ratings (IP rating, temperature range)

3. **Safety Documentation**

   * Risk assessment document
   * CE/UL certification (if applicable)
   * Safety data sheets for any hazardous materials

Software Deliverables
---------------------

For URCap and software integrations:

1. **URCap Package**

   * Signed ``.urcap`` file for PolyScope 5 (if applicable)
   * Container image for PolyScope X (if applicable)
   * Source code access (for review purposes)

2. **Integration Materials**

   * Sample programs demonstrating key features
   * URScript function library (if applicable)
   * REST API documentation (if applicable)

3. **Configuration Files**

   * Default configuration settings
   * Robot installation files
   * Safety configuration templates

Documentation Deliverables
--------------------------

Comprehensive documentation is essential:

1. **User Manual**

   * Installation instructions
   * Setup and configuration guide
   * Operation procedures
   * Troubleshooting guide
   * Safety warnings and precautions

2. **Quick Start Guide**

   * Single-page setup overview
   * Basic operation steps
   * QR code linking to full documentation

3. **Technical Documentation**

   * API reference (if applicable)
   * Communication protocol specifications
   * Integration architecture diagram

4. **Marketing Materials**

   * Product images (high resolution)
   * Product description for UR+ Showroom
   * Demo video (recommended)

Submission Format
-----------------

* All documents in PDF format
* Source files in editable format (Word, InDesign, etc.)
* Software packages with version numbers
* Organized folder structure

.. note::
   Incomplete deliverables will delay the certification process. Ensure all items are ready before submission.

Version Control
---------------

Partners must maintain version control for all deliverables:

* Use semantic versioning (e.g., 1.0.0)
* Document changes between versions
* Maintain compatibility matrix with PolyScope versions
